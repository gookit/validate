| **#259** | Embedded structs not validated properly | ✅ **已修** | 匿名嵌入现始终级联、字段被验证（T10 豁免）。报告里 perm2 仍报错是设计行为：内层 `required` 独立求值，不被外层 `required_if` 门控 | `TestIssue_259_v2` |

| #327 | 未指定 required 时取不到字段值 | ✅ 非 bug（设计） | 只有"参与校验且通过"的字段进 safeData / 可被 BindStruct 绑定；非 required 规则 + 非空值能正常绑定；完全无规则字段才取不到。`AddMessages` 不写 required 项不影响其它字段消息 | `TestIssue_327_v2` |
| #235 | 如何验证 struct 内的 slice-of-struct | 🟡 partial | 自定义验证器 workaround 在 v2.0 完全可用（需给字段加 validate tag 触发级联）；路径式 `required_with:Nodes..Location` 跨元素引用仍失效 | `TestIssue_235_v2` |
| **#217** | Nested resources evaluated differently | ✅ **已修**（`da6eda7`） | 切片元素里 `*bool→false` 曾被 `required` 误判为空：`data_source.go` 子结构体导航循环把叶子指针也解引用成 `bool(false)`（`IsEmpty==true`）。修复让叶子节点保留非 nil 指针（与顶层路径一致），中间节点仍解引用导航 | `TestIssue_217_v2` |
| **#283** | Scenes does not work in slices | ✅ **已修**（`1fdb557`，仅 `.*`） | 场景通配 `Tags.*.Id` 现可命中级联生成的 `Tags.0.Id`：`sceneFieldMap` 把含 `.*` 的项收进 `sceneWildcards`，`isNotNeedToCheck` 把字段名数字索引段规范化为 `*` 再比对。无索引 `Tags.Id` 因歧义按设计不做 | `TestIssue_283_v2` |
| **#314** | Scene with empty validation rules now fails (回退) | ✅ **已修**（`d1031d6`） | `"None": []string{""}` 本应"不校验任何字段"，但 `sceneFieldMap()` 把 `""` 当普通键，`isNotNeedToCheck` 里 `strings.Join(fields[0:0],".")` 得 `""` 恰好命中。修复：`sceneFieldMap` 跳过空字段但保留非 nil map；`isNotNeedToCheck` 用 nil/非nil 区分"未设场景(校验全部)"与"场景激活但无字段(都不校验)" | `TestIssue_314_v2` |
| **#262** | filter 无法对切片中元素应用 | ✅ **已修**（`6e600ed`） | `FilterRule.Apply` 对**内部**通配（`.*` 后还有字段）逐元素 apply filter，结果写回 `filteredData[通配key]` 与验证器衔接；**末尾** `.*`（如 `Domains.*`）仍走整切片以兼容 `trimStrings` 等切片级 filter（#172）。范围：Map/JSON + 单级 `.*` | `TestIssue_262_v2` |
