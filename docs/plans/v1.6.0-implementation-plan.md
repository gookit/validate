# validate v1.6.0 实施计划

> **目标版本**: v1.6.0（基于 v1.5.7）
> **性质**: 保持公共 API 兼容的内部性能重构（升级零改动，只是更快）
> **设计依据**: `docs/validate-v2-design.md`（rev.3，§6.1 P0–P6）
> **Go 版本**: 维持 `go 1.19` 不变
> **状态**: 待执行
> **创建**: 2026-06-03

---

## 0. 总览

把 `docs/validate-v2-design.md` §6.1 的 P0–P6 落地为 v1.6.0。核心三件事：
1. **TypeMeta**：缓存 struct 类型级反射元数据（字段索引、tag、预解析规则模板）。
2. **FieldValue**：贯穿管线、携带 `reflect.Value` 的值载体，消除 `reflect.Value→Interface()→reflect.ValueOf` 往返。
3. **管线拆分 + 实现下沉 internal**，提升可维护性。

**红线（每阶段都必须满足）**：
- 现有全部测试零回归（`issues_test.go`/`issues_x2_test.go`/`validation_test.go`/`validators_test.go`/`data_source_test.go`/`morecase_test.go` 等）。
- 公共 API（包级函数、`Validation` 公共方法、`DataFace` 接口、内置验证器函数签名、struct tag 语义）**不变**。
- 基准不回退（以 P0 基线为准）。

> ⚠️ 破坏性内容（验证器统一 `reflect.Value` 入参、`DataFace` 扩展、注册自定义类型、`/v2`、升 Go 1.21）**不属于本计划**，留待 1.6.0 稳定后的 v2.0.0。见设计文档 §6.0。

### P1 — FieldValue 值载体 🟢低风险

**目标**：引入携带 `reflect.Value` 的值载体，先在热点函数内部替换重复 `reflect.ValueOf`，**对外行为完全不变**。

**关键设计（见设计 §4.3）**
```go
// 新文件：fieldvalue.go（根包；P4 再考虑迁 internal）
type FieldValue struct {
    name string
    src  any
    rv   reflect.Value  // 懒：reflect.ValueOf(src)；若来源已有 reflect.Value 则直接塞入
    rt   reflect.Type   // 懒
    real reflect.Value  // 懒：reflect.Indirect(rv)
    rvInit, realInit bool
    zero, empty int8    // 0未知 1是 2否（懒）
}
func (f *FieldValue) RV() reflect.Value   // 懒构造并缓存
func (f *FieldValue) Real() reflect.Value // 去指针，统一指针语义
func (f *FieldValue) IsZero() bool        // 懒
func (f *FieldValue) IsEmpty() bool       // 懒，复用 RV()，替代 IsEmpty(any) 的再反射
```

**任务**
- [ ] 新建 `fieldvalue.go`：`FieldValue` 结构 + `RV/Real/IsZero/IsEmpty/Interface` 方法（懒求值）
- [ ] 提供两种构造：`newFieldValue(name, src)`（map/form 用，src 为普通 any）/ `newFieldValueRV(name, rv)`（struct 用，直接塞已有 reflect.Value，**不调用 `.Interface()`**）
- [ ] 新建 `fieldvalue_test.go`：覆盖指针 / nil 指针 / 零值 / 空 slice/map / 普通值的 `IsZero/IsEmpty/Real` 语义，与现有 `IsEmpty(any)`、`TryGet` 返回的 zero 对齐
- [ ] 在 `callValidatorValue`（validating.go:542）内部：用传入的 `FieldValue.RV()` 替代 `reflect.ValueOf(val)`（validating.go:549），指针解引用改用 `Real()`
- [ ] 在 `valueValidate`（validating.go:300）内部：复用 `FieldValue.RV()`，去掉重复 `reflect.ValueOf(val)`
- [ ] 集中指针/零值逻辑：标注/收敛 `convValAsFuncValArgType`、`removeValuePtr` 的散用点（不删公共函数，内部改走 FieldValue）

**注意 / 坑**
- `IsEmpty` 是公共函数 `validate.IsEmpty(any)`（validators.go:542），**不能动签名**；`FieldValue.IsEmpty()` 内部复用其底层 `ValueIsEmpty(reflect.Value)`（util.go:185）避免再 `reflect.ValueOf`。
- 避免 import 环：FieldValue 暂放根包即可直接调 `ValueIsEmpty`。
- typed-nil vs untyped-nil：保持现有 `nilRVal`（util.go:23）处理路径，`FieldValue.RV()` 对 invalid 值返回 `nilRVal`。

**验收门禁**：全量测试绿；`BenchmarkStructFlat` 的 allocs/op 较 P0 **下降**（少 1~2 次 reflect.ValueOf/装箱）；其余基准不回退。
**提交**：`perf(fieldval): introduce FieldValue to remove reflect round-trip`

### P2 — 类型元数据缓存 TypeMeta 🟡中风险

**目标**：同一 struct 类型的字段索引 / kind / tag 只构建一次并缓存；字段访问由 `FieldByName`（O(n)）改 `FieldByIndex`（O(1)）。

**关键设计（见设计 §4.2 / §4.6）**
```go
// cache.go（替换现有全注释内容）
type FieldMeta struct {
    Index   []int        // FieldByIndex 路径
    Name, Path string
    Kind   reflect.Kind  // 去指针后
    IsPtr  bool
    Elem   elemClass     // leaf / structKind / sliceOfStruct / mapOfStruct...
    ValidateRule, FilterRule, OutputName, Label string
    Messages map[string]string
}
type TypeMeta struct {
    Type   reflect.Type
    Fields []*FieldMeta
    byName map[string]*FieldMeta
    // P3 再填充：StaticRules / DynamicFields
    Labels, Outputs map[string]string
    ImplConfig, ImplTranslates, ImplMessages bool
}
type typeKey struct { rt reflect.Type; tagVer uint32 }
var typeCache sync.Map  // map[typeKey]*TypeMeta
func getTypeMeta(rt reflect.Type) *TypeMeta  // 命中即返回；未命中构建后存入
```

**任务**
- [ ] `cache.go`：实现 `FieldMeta`/`TypeMeta`/`typeKey` + `getTypeMeta`（仅类型级遍历 `rt.Field(i)`，**不碰具体值**）
- [ ] tag 配置版本：`validate.go` 的 `Config()`/`ResetOption()` 内 `tagVer++`（用 `sync/atomic`）；`getTypeMeta` 用当前 `tagVer` 组 key → 配置变更自动失效旧缓存
- [ ] 新增公共方法 `validate.ResetTypeCache()`（清空 `typeCache`，供测试/特殊场景）
- [ ] `StructData`：增加 `meta *TypeMeta` 字段；`FromStruct`（validate.go:343）构建/取缓存 meta
- [ ] `StructData.TryGet`（data_source.go:530）/`Set`（:639）：顶层字段访问改用 `value.FieldByIndex(meta.byName[field].Index)`，替代 `FieldByName`
- [ ] 一次性接口判定：`Create()`（data_source.go:244-262）的 `Implements(cvFaceType/ftFaceType/cmFaceType)` 结果存入 `TypeMeta`，免每实例三次 `Implements`
- [ ] 新建 `cache_test.go`：
  - [ ] 命中缓存返回同一 `*TypeMeta`（`assert.Same`）
  - [ ] `Config(ValidateTag)` 变更后 `getTypeMeta` 失效、重建
  - [ ] `*T` 与 `T` 经 `reflects.Elem` 后命中同一 elem 类型缓存
  - [ ] `go test -race` 并发构建无竞态

**注意 / 坑**
- 匿名 / 私有字段：`FieldByIndex` 对非导出字段会 panic（`reflect: ...`）；保留现有 `fieldNames` 分类（top/anonymous/sub，data_source.go:32-36）与 `ValidatePrivateFields` 分支，`FieldByIndex` 仅用于可导出路径，非导出仍走原逻辑。
- 嵌套 `FieldByIndex` 遇 nil 指针中间层会 panic → 沿用现有"按需初始化指针"（data_source.go:430/657）或 `FieldByIndexErr` 风格的安全访问。
- `time.Time` 等需排除递归（沿用 `timeType` 判断 data_source.go:219）。

**验收门禁**：全量测试绿 + `-race` 绿；`BenchmarkStructFlat`/`Nested` 的 ns/op 较 P0 显著下降（同类型二次验证收益最大）；`BenchmarkMapValidate` 持平（±5%）。
**提交**（按功能点拆）：
- `refactor(meta): add TypeMeta/FieldMeta and sync.Map type cache`
- `perf(struct): access fields via FieldByIndex using cached meta`
- `feat(cache): add tagVer invalidation and ResetTypeCache()`

### P3 — 规则模板缓存 RuleTpl + build/instantiate 两段式 🟡中风险

**目标**：把规则收集拆成"类型级构建（缓存）"+"值级实例化"，并把规则字符串解析、别名解析、参数类型转换前移到构建期（最大增益点之一）。

**关键设计（见设计 §4.4）**
```go
type RuleTpl struct {
    Path, Validator, RealName string
    NotRequired bool
    Args        []any      // 构建期已按验证器签名转换好，运行时不可变
    FuncMeta    *funcMeta  // 全局/内置验证器构建期绑定；struct 方法验证器留空、运行时解析
    FilterRule  string
}
// TypeMeta 补充：
//   StaticRules   []*RuleTpl   // 路径恒定（leaf、struct-of-struct）
//   DynamicFields []*FieldMeta // slice/map-of-struct，需按值展开
```
拆分 `parseRulesFromTag`（data_source.go:270）：
- `build(rt)`：只遍历**类型**，解析全部 tag → 生成 `StaticRules` + 翻译表；slice/map-of-struct 记入 `DynamicFields`（递归缓存元素类型 meta）。
- `instantiate(value, v)`：挂载 `StaticRules`（零反射）；仅对 `DynamicFields` 按真实值展开元素路径（`items.0.name`...）。

**任务**
- [ ] `cache.go`/新 `struct_rules.go`：`RuleTpl` + `build` 阶段（tag 解析、`splitRules`/`ValidatorName` 别名、`parseArgString`、参数类型转换前移）
- [ ] 构建期参数转换：复用现 `convertArgsType`（validating.go:472）逻辑，但作用于"字面参数 + 验证器签名"（皆类型级），结果存 `RuleTpl.Args`
- [ ] 移除运行时 in-place 改写参数（validating.go:530 `args[i]=...`）——改为读不可变 `RuleTpl.Args`
- [ ] `StructData.Create`（data_source.go:234）改走 `build`(缓存) + `instantiate`
- [ ] `instantiate` 平移现有 slice/map 元素展开逻辑（含 `CheckSubOnParentMarked` data_source.go:359、`IsSimpleKind` 跳过 :377、nil 指针初始化 :430、匿名字段、map key 格式化 :407-425）
- [ ] **回归对照测试** `regression_compat_test.go`：对一组覆盖各形态的 struct，断言 v1.6 生成的规则集合（path + validator + realName + args）与改造前**逐条一致**；翻译表（label/output/message）一致
- [ ] 把拆出的规则收集代码放新文件，使 `data_source.go` 回落到 ≤1000 行

**注意 / 坑**
- `RuleTpl.FuncMeta` 仅对**构建时已存在**的内置/全局验证器绑定；自定义/同名覆盖/struct 方法验证器（`StructData.FuncValue` data_source.go:750）一律运行时解析（同现状 validation.go:243-254 回退路径）。
- 共享不可变性：`StaticRules` 进缓存后被多实例/多 goroutine 共享，**任何运行时变异都禁止**；实例期如需 per-instance 状态（如 scene、消息覆盖），在 Validation 侧维护，不写回模板。
- `Val()`（value.go）路径不经 struct，不受影响，但顺带确认其 `buildRule` 仍可用。
- 边界：空 struct、仅 filter 无 validate、仅 default、`safe`/`-` 字段。

**验收门禁**：全量测试绿；`regression_compat_test.go` 规则集合一致；首次解析 allocs/op 不高于 P0 +15%；同类型二次验证 ns/op 进一步下降。
**提交**（拆分）：
- `perf(rule): pre-parse rule templates (alias+args) at type-build time`
- `refactor(struct): split rule collection into build/instantiate`
- `test(compat): assert v1.6 rule set matches v1.5 baseline`

### P4 — 验证管线拆分 + 下沉 internal 🟡中风险

**目标**：拆分巨函数提升可维护性（痛点 D）；把纯内部机制迁入 `internal/`（痛点 E）。**行为零回归**。

**关键设计（见设计 §4.5 / §4.1）**
- `Rule.Apply` 拆为：`skipByScene` → `stage`(default/filter/skip-empty 前置) → `check` → `saveSafe`/`addError`。
- `check(fv)` 分支独立成函数：safe 短路 / `required*` 直调 / `.*` 通配符 slice 校验 / 普通验证器（内置 switch 快路径 + 自定义反射，复用 `fv.RV()`）。
- `buildArgIn(fv, tpl)` 统一构造验证器入参，消除每次 `reflect.ValueOf(arg/data)`（validating.go:497/564/568）。

**任务**
- [ ] 拆分 `Rule.Apply`（validating.go:85-192）为多个职责单一方法
- [ ] 拆分 `valueValidate`（validating.go:250-371）：safe / required / 通配符 slice / 普通验证器 各自成函数；`flatSlice` 多层展开、map 通配符父切片长度对比（`getParentSliceLen` :593）逻辑原样平移
- [ ] `buildArgIn` 封装入参构造，`data`/`val` 位用 `fv.RV()`，args 用 `RuleTpl.Args`
- [ ] 新建 `internal/` 子包并迁移：`internal/meta`(TypeMeta/FieldMeta/RuleTpl)、`internal/fieldval`(FieldValue)、`internal/exec`(管线)
- [ ] 处理 import 环：`IsEmpty/ValueIsEmpty` 的底层 reflect 原语抽到 `internal/reflectx`，根包 `validate.IsEmpty` 改为再导出（**公共签名不变**）；internal 子包单向依赖，根包依赖 internal
- [ ] 根包只保留：公共 type/interface、包级入口、对 internal 的薄封装

**注意 / 坑**
- 这是"大搬家"，**务必小步提交**：先在根包内拆函数（不动包结构）→ 全测绿 → 再逐个迁 internal → 每迁一个包跑全测。
- `internal/` 对外不可见，迁移不破坏 API；但要确保根包导出符号（如 `Rule`、`Validation` 字段可见性）不变。
- 若 import 环难解，允许 FieldValue/TypeMeta 暂留根包（设计 §4.1 已许可"先根包内"），internal 化作为 best-effort，不阻塞发版。

**验收门禁**：全量测试绿 + `-race` 绿；基准较 P3 不回退；`go vet`/`golangci-lint` 通过；`data_source.go` ≤1000 行。
**提交**（拆分）：
- `refactor(exec): split Rule.Apply into single-responsibility steps`
- `refactor(exec): decompose valueValidate (safe/required/wildcard/normal)`
- `refactor(internal): move meta/fieldval/exec into internal packages`

### P5 — 可选 Factory / Pool + emptyV 并发安全 🟢低风险

**目标**：为"同类型大批量校验"提供可选的实例复用；修复 `Val()` 共享 `emptyV` 的并发隐患。**默认行为不变**。

**关键设计（见设计 §4.7）**
```go
f := validate.NewFactory()
v := f.Struct(&user)   // pool 取 Validation + 命中 TypeMeta 缓存
defer v.Release()      // Reset 后归还
v.Validate()
```

**任务**
- [ ] 新增 `Factory`：`NewFactory()` + `Struct/Map(...)` + `(*Validation).Release()`（`Reset` 全字段后 `pool.Put`）
- [ ] `Validation.Reset`（validation.go:134）补全：确保 safeData/filteredData/Errors/rules/optionals/scene/sceneFields 等全部清理干净
- [ ] `value.go` 的 `emptyV`（value.go:11）并发安全改造：每次 `Val()` 用栈对象或独立 pool 取，避免全局共享实例被并发写
- [ ] 测试：`-race` 下并发 `Factory` 复用、并发 `Val()`；内存复用不串数据；（可选）`go test -run xxx -memprofile` 检查无泄漏

**注意 / 坑**
- 池化是 opt-in：默认 `validate.Struct/Map` 生命周期与现状一致，不得引入隐式池化。
- `Release` 后禁止再用该 `v`（文档注明）。

**验收门禁**：全量测试绿 + `-race` 绿；Factory 路径基准优于默认路径；`Val()` 并发无数据竞争。
**提交**：
- `feat(factory): add optional Factory with pooled Validation reuse`
- `fix(value): make Val() shared validation concurrency-safe`

## 5. 风险与回滚

| 风险 | 阶段 | 缓解 / 回滚 |
|---|---|---|
| build/instantiate 拆分遗漏分支，规则集合不一致 | P3 | `regression_compat_test.go` 守护；不一致即回滚该提交 |
| `FieldByIndex` 对匿名/私有字段 panic | P2 | 保留 `fieldNames` 分类，非导出走原路径；用例覆盖 `ValidatePrivateFields` |
| 共享模板被运行时变异 → 并发串数据 | P3/P5 | 模板构建后只读；`-race` 跑并发基准 |
| internal 迁移引发 import 环 | P4 | 小步提交；难解则 FieldValue/TypeMeta 暂留根包（设计已许可） |
| 池化 Reset 不彻底串数据 | P5 | opt-in、非默认；`-race` + Reset 全字段核对 |

**回滚策略**：每阶段独立提交，出问题回滚到上一阶段末尾的绿色提交即可；不同阶段互不阻塞发版下限——即使只完成到 P3，已具备可发布的性能收益。
