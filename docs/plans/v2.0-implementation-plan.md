## 0. 总览

v1.6.0 已完成**保持公共 API 兼容**的内部性能重构（扁平 struct -78% 耗时 / -74% 分配）。
v2.0.0 在此之上做**破坏性**精简，核心五件事：

1. **删运行期 `convertArgsType`**：比较类（Gt/Lt/Min/Max）的 `reflect.Value`/kind-aware 统一 v1.6.0 已由 `valueCompare` 落地；本次把残余的 args 类型转换**前移到构建期**（不可变 `RuleTpl.Args`），删除运行期 `convertArgsType`，顺带统一 `Between`（int64→any）并修 `val.(string)` 类验证器的 panic。详见 T3 范围澄清。
2. ~~`DataFace` 扩展 + 消除源头 `Interface()` 装箱~~ → **已 profile 评估废弃**（装箱守恒、净零收益；`BindStruct/SafeVal` 是 Validation 方法非数据源）。T4 **重定义**为「共享不可变静态模板规则」（消除 `cloneRule` 27% 大头，零 API 破坏）。详见 T4。
3. **注册自定义类型**：开放 `can register custom type` 能力。
4. **清理历史包袱**：删 deprecated（`ValueLen` 等）、升 Go 1.21、模块升 `/v2`。
5. **opt-in Factory/Pool**：落地 v1.6.0 延后的 `Struct/Map` 池化工厂。

**破坏性边界（明确告知用户）**：
- 直接调用内置验证器函数（仅 `Between`：`int64`→`any`）的代码需改签名；删 deprecated `ValueLen`。
- import 路径需带 `/v2`；最低 Go 版本升至 1.21。
- ~~自定义 `DataFace` 实现者~~ —— T4 重定义后 **`DataFace` 不再破坏**，自定义实现者不受影响。

**红线（每阶段都必须满足）**：
- **行为零回归**：v1.6.0 的 `regression_compat_test.go` 黄金网 + 全量测试（`issues_test.go`/`issues_x2_test.go`/`validation_test.go`/`validators_test.go`/`data_source_test.go`/`morecase_test.go`）须全绿。
  - ⚠️ 例外：T3（验证器签名）会**主动改变** golden 的 `%#v` 形态（args 类型语义变了）——届时**同步更新 golden 期望值**并在提交说明中标注，而非视作回归。
- **性能不回退**：以 v1.6.0 终态（`docs/plans/p6-final.txt`）为基线；理想是 T3/T4 进一步降 allocs。
- 每个**破坏性变更**当场记入 §5 破坏性清单（喂给最终迁移指南）。

> 📌 设计 §6.0 原话："v2.0.0 的破坏性内容待 1.6.0 稳定发布后另立设计文档"。本次按用户决策**不另写设计文档**，范围以本计划 §3 为准；其中 T5（注册自定义类型）API 形态尚未细化，执行前需先定（见该阶段「前置决策」）。

### T3 — 构建期预转换 args + 删运行期 `convertArgsType`（含 Between 统一 + 修 string 断言 panic）🟡中风险

> **范围澄清（实现期核对后修正——原"统一为 any"是过时/过度简化）**：
> - CHANGELOG V2-TODO #1 的「old」示例 `Gt(val any, dstVal int64)` **已在 v1.6.0 落地**：现 `Gt(val, minVal any)` 委托 `valueCompare`（util.go:254，已 kind-aware：string→`strutil.Compare`、int/uint/float→`mathutil.Compare`）。Gt/Lt/Min/Max/Enum/NotIn **无需再动**。
> - `convertArgsType`（validating.go:514）有早退「单 `any` 实参即跳过」，故它的运行期开销**只剩**给「有具体类型实参」的验证器：`Between(int64,int64)`、`Length/MinLength/MaxLength(int)`、`RuneLength/StringLength(int)`、`IsInt(...int64)`、`IsString(...int)`——因 switch 要 `args[0].(int64)`/`.(int)`。
> - **决策：不盲目把这些实参改 `any`**（长度→`any` 是语义退步）。按语义保留具体类型，把「string→int64/int」转换从**运行期 convertArgsType** 前移到**构建期 `RuleTpl.Args`**（即 v1.6.0 P3 点名"留待 v2"的那件事），同样删掉运行期转换，且**不破坏签名、不丢类型安全**。
> - **不用泛型**：分派是 `switch{ Between(val, args[0].(int64), …) }`，args 是运行时 `[]any`，无法用运行时值实例化泛型；泛型函数也存不进 `funcMeta.fv` 做反射分派。类型安全的直接调用如需要，另起 opt-in 泛型 predicate 辅助包作 follow-up，不进 v2 核心。

**目标**：删除运行期 `convertArgsType`（args 转换前移到构建期、存入不可变 `RuleTpl.Args`）；顺带修两处真实问题——`Between` 的 `int64` 不一致（→ `any`，与 Gt 统一、支持 float 阈值）、`val.(string)` 类验证器对非字符串字段 panic。

**关键设计（路线 2：转换前移 + 保留具体类型）**
```go
// 公共签名按语义保留具体类型，仅 Between 统一为 any（唯一签名破坏点）
func Between(val, minVal, maxVal any) bool   // 原 (val any, min,max int64)，与 Gt 一致走 valueCompare
func Length(val any, wantLen int) bool       // 长度天然 int，不动
func Regexp(str, pattern string) bool        // 模式天然 string，不动

// args 转换前移：构建 RuleTpl 时按验证器签名把 tag 字符串转成目标类型，存入不可变 Args
// 运行期 switch 仍 args[0].(int) / .(int64)，但 args 已在构建期转好 → convertArgsType 整体删除

// 修 panic：分派前对 val.(string) 类验证器做安全转换（非字符串 → 转 string 或直接判 false，不 panic）
```

### T4 — 共享不可变静态模板规则（免每实例 cloneRule）🟡中风险

> **重定义（profile 驱动——原 DataFace 设计已评估废弃）**：
> 原 T4 是「`DataFace.TryGet` 返回 `reflect.Value` 载体 + 消除源头 `Interface()` 装箱 + `BindStruct/SafeVal` 进接口」。**实测后否决**，两条硬理由：
> 1. **装箱守恒**：T3 选了路线 2（验证器保持 `any` 入参），struct 字段值迟早必须装箱成 `any` 才能喂验证器。消除源头装箱只是把那一次 `Interface()` 从 `TryGet` 挪到验证器调用处，**净分配为零**。pprof 实测 `reflect.packEface`（装箱）仅占 StructFlat 的 **6.4%**，且 `safeData`/`filteredData` 已缓存跨规则值。
> 2. **`BindStruct/SafeVal` 是 `Validation` 的方法**（validation.go:496/508），**不是数据源的**——设计 §6.0「进 DataFace 接口」定义不清、无落点。
> 结论：原 T4 = 破坏自定义 `DataFace` 实现者 + 零收益，违背最小破坏原则，**不做**。

**目标（profile 实锤的真实杠杆）**：pprof 显示 `cloneRule` 是 StructFlat 31 allocs 的**最大单一来源（27%）**——`instantiateStatic` 每次校验逐条克隆模板 Rule 结构。而 Rule 校验期**完全只读**（全仓库确认字段写入只在构建期）、`argsReady` 规则的 args 已不可变（T3-3b）⇒ 可**直接共享模板 `*Rule` 指针**，免克隆。

**关键设计（加法式、零 API 破坏）**
```go
// instantiateStatic：保留 v 自己的 backing 数组,但 argsReady 规则共享模板指针
v.rules = make([]*Rule, 0, len(tpl.rules))
for _, tr := range tpl.rules {
    if tr.argsReady {
        v.rules = append(v.rules, tr)        // 共享不可变模板规则,免 cloneRule
    } else {
        v.rules = append(v.rules, cloneRule(tr)) // 非 argsReady 运行期改 args,仍克隆
    }
}
```
- 安全性同 T3-3b：Rule 校验期只读、`v.rules` 保留自己的 backing（Reset `[:0]`/复用/AddRule 不碰模板）、append 新规则创建新指针不改旧的。
- 仅作用于**静态结构体模板**;Map/Form/动态路径不受影响（持平）。

### T5 — 注册自定义类型 🟡中风险

**目标**：开放 `can register custom type` 能力（CHANGELOG V2-TODO），让 validate 知道如何处理自定义类型（如 `sql.NullString`/`decimal.Decimal`/`null.Int`/枚举包装类型）——核心是教它从自定义类型**提取出可校验的底层值**。

**API 设计（已定：值提取器模式，对标 go-playground/validator 的 `RegisterCustomTypeFunc`）**
```go
// CustomTypeFunc 从自定义类型字段提取 validate 实际要校验的底层值。
// 返回 nil 表示"视为空/未设置"(required 失败)。
type CustomTypeFunc func(field reflect.Value) any

// AddCustomType 为给定样例类型注册提取器(命名与 AddValidator 一致)。
//   validate.AddCustomType(func(f reflect.Value) any {
//       if ns := f.Interface().(sql.NullString); ns.Valid { return ns.String }
//       return nil
//   }, sql.NullString{})
func AddCustomType(fn CustomTypeFunc, types ...any)
```

**设计决策（已定）**
| 点 | 决策 | 理由 |
|---|---|---|
| 形态 | **单一值提取器**,非多方法 handler 接口 | 提取后的底层值天然走现有 `valueCompare`/`IsEmpty`/`Length`/string 路径——一个 extractor 覆盖比较/空值/长度全部,无需 `IsEmpty/IsZero/Compare/Len` 各写一遍 |
| 命名 | `AddCustomType` + `CustomTypeFunc`(可加 `AddCustomTypes` 复数别名) | 与现有 `AddValidator(s)` 一致 |
| 注册表 | 包级 `customTypes map[reflect.Type]CustomTypeFunc` + `RWMutex`(或 `sync.Map`) | 与 `validatorMetas` 同风格;注册通常在 init 期、读多写少 |
| **零开销快路径** | 用原子计数/`len==0` 判断：**未注册任何自定义类型时完全跳过查找** | 绝大多数用户不注册→默认路径零成本(基准必须持平) |
| 接入点 | `fieldValue` 构造处(`newFieldValue`/`newFieldValueRV`):若 `src`/`rv` 类型命中注册表,提取并替换 effective `src` | 一处接入,下游比较/空值/长度/string 全部透明看到提取后的值 |
| 语义 | extractor 返回 nil→空(required 失败);非 nil→以该值校验 | 对标 go-playground;`sql.NullString{Valid:false}`→nil→空 |
| 缓存交互 | v2.0 先做**运行期类型查找**(快路径守护),不动 typeMeta | 注册可运行期发生;`fieldMeta.isCustomType` 标志缓存留作后续可选优化 |

**任务**
- [x] `register_type.go`:`CustomTypeFunc` + `customTypes sync.Map`(无锁读)+ `hasCustomTypes atomic.Bool` 门控 + `AddCustomType` + `ResetCustomTypes` + `resolveCustomType(val)(any,bool)`
- [x] 接入 `valueValidate`（**safe 后、required 快路径前**）+ `validateWildcardSlice`（`.*` 元素;提取为 nil 时避免 invalid Value `Interface()` panic）
- [x] `register_type_test.go`:`sql.NullString` required/minLen + 自定义 `Money` min/max/between + 未命中不变 + reset 失效 + 并发 + nil 守卫 + wildcard
- [ ] README 示例（留 T8 收尾）

**实现对账（偏离记录）**
| 计划 | 实际 | 原因 |
|---|---|---|
| 接入 `fieldValue` 构造 | 接入 `valueValidate`（required 快路径**之前**） | `required` 走 `v.Required(field,val)` 早于 fieldValue 构造;接 fieldValue 则 required/空值看不到提取值。在 required 前提取覆盖全部路径 |
| 注册表 RWMutex | `sync.Map` | 读多写少、热路径无锁读更优 |
| 直接调 `resolveCustomType` | **门控 `hasCustomTypes.Load()` 内联在调用点** | `resolveCustomType` cost 266 不可内联,直接调致 +2% 回退;内联门控后未注册仅一次 atomic load |

**验收门禁**：✅ golden `count=3` 不变;全量 + race 绿（自定义类型 `-race -count=2` 亦绿）;**未注册时 StructFlat 23 / Map 72 allocs 零回退**(主控独立复测确认)。
