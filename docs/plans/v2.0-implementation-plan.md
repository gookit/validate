## 0. 总览

v1.6.0 已完成**保持公共 API 兼容**的内部性能重构（扁平 struct -78% 耗时 / -74% 分配）。
v2.0.0 在此之上做**破坏性**精简，核心五件事：

1. **删运行期 `convertArgsType`**：比较类（Gt/Lt/Min/Max）的 `reflect.Value`/kind-aware 统一 v1.6.0 已由 `valueCompare` 落地；本次把残余的 args 类型转换**前移到构建期**（不可变 `RuleTpl.Args`），删除运行期 `convertArgsType`，顺带统一 `Between`（int64→any）并修 `val.(string)` 类验证器的 panic。详见 V3 范围澄清。
2. **`DataFace` 扩展 + 消除源头 `Interface()` 装箱**：`TryGet` 返回携带 `reflect.Value` 的载体，落地设计 §4.3 完整版（v1.6.0 因属破坏性而搁置的最后一跳）。
3. **注册自定义类型**：开放 `can register custom type` 能力。
4. **清理历史包袱**：删 deprecated（`ValueLen` 等）、升 Go 1.21、模块升 `/v2`。
5. **opt-in Factory/Pool**：落地 v1.6.0 延后的 `Struct/Map` 池化工厂。

**破坏性边界（明确告知用户）**：
- 直接调用内置验证器函数（如 `validate.Gt(x, int64(5))`）的代码需改签名。
- 自定义 `DataFace` 实现者需实现扩展后的接口。
- import 路径需带 `/v2`；最低 Go 版本升至 1.21。

**红线（每阶段都必须满足）**：
- **行为零回归**：v1.6.0 的 `regression_compat_test.go` 黄金网 + 全量测试（`issues_test.go`/`issues_x2_test.go`/`validation_test.go`/`validators_test.go`/`data_source_test.go`/`morecase_test.go`）须全绿。
  - ⚠️ 例外：V3（验证器签名）会**主动改变** golden 的 `%#v` 形态（args 类型语义变了）——届时**同步更新 golden 期望值**并在提交说明中标注，而非视作回归。
- **性能不回退**：以 v1.6.0 终态（`docs/plans/p6-final.txt`）为基线；理想是 V3/V4 进一步降 allocs。
- 每个**破坏性变更**当场记入 §5 破坏性清单（喂给最终迁移指南）。

> 📌 设计 §6.0 原话："v2.0.0 的破坏性内容待 1.6.0 稳定发布后另立设计文档"。本次按用户决策**不另写设计文档**，范围以本计划 §3 为准；其中 V5（注册自定义类型）API 形态尚未细化，执行前需先定（见该阶段「前置决策」）。

### V3 — 构建期预转换 args + 删运行期 `convertArgsType`（含 Between 统一 + 修 string 断言 panic）🟡中风险

> **范围澄清（实现期核对后修正——原"统一为 any"是过时/过度简化）**：
> - CHANGELOG V2-TODO #1 的「old」示例 `Gt(val any, dstVal int64)` **已在 v1.6.0 落地**：现 `Gt(val, minVal any)` 委托 `valueCompare`（util.go:254，已 kind-aware：string→`strutil.Compare`、int/uint/float→`mathutil.Compare`）。Gt/Lt/Min/Max/Enum/NotIn **无需再动**。
> - `convertArgsType`（validating.go:514）有早退「单 `any` 实参即跳过」，故它的运行期开销**只剩**给「有具体类型实参」的验证器：`Between(int64,int64)`、`Length/MinLength/MaxLength(int)`、`RuneLength/StringLength(int)`、`IsInt(...int64)`、`IsString(...int)`——因 switch 要 `args[0].(int64)`/`.(int)`。
> - **决策：不盲目把这些实参改 `any`**（长度→`any` 是语义退步）。按语义保留具体类型，把「string→int64/int」转换从**运行期 convertArgsType** 前移到**构建期 `RuleTpl.Args`**（即 v1.6.0 P3 点名"留待 v2"的那件事），同样删掉运行期转换，且**不破坏签名、不丢类型安全**。
> - **不用泛型**：分派是 `switch{ Between(val, args[0].(int64), …) }`，args 是运行时 `[]any`，无法用运行时值实例化泛型；泛型函数也存不进 `funcMeta.fv` 做反射分派。类型安全的直接调用如需要，另起 opt-in 泛型 predicate 辅助包作 follow-up，不进 v2 核心。

**目标**：删除运行期 `convertArgsType`（args 转换前移到构建期、存入不可变 `RuleTpl.Args`）；顺带修两处真实问题——`Between` 的 `int64` 不一致（→ `any`，与 Gt 统一、支持 float 阈值）、`val.(string)` 类验证器对非字符串字段 panic。

**关键设计（路线 2：转换前移 + 保留具体类型）**
```go
// 公共签名按语义保留具体类型，仅 Between 统一为 any（唯一签名破坏点）
func Between(val, minVal, maxVal any) bool   // 原 (val any, min,max int64)，与 Gt 一致走 valueCompare
func Length(val any, wantLen int) bool       // 长度天然 int，不动
func Regexp(str, pattern string) bool        // 模式天然 string，不动

// args 转换前移：构建 RuleTpl 时按验证器签名把 tag 字符串转成目标类型，存入不可变 Args
// 运行期 switch 仍 args[0].(int) / .(int64)，但 args 已在构建期转好 → convertArgsType 整体删除

// 修 panic：分派前对 val.(string) 类验证器做安全转换（非字符串 → 转 string 或直接判 false，不 panic）
```

**任务**
- [ ] 构建期预转换：`RuleTpl.Args` 在类型构建期按验证器 `funcMeta` 签名把字符串实参转成目标类型（int/int64…），不可变；复用现 `convertArgsType` 的转换逻辑，只是**搬到构建期、对类型级缓存做一次**
- [ ] 删除运行期 `convertArgsType`（validating.go:514 定义 + :304 调用点）；switch 的 `.(int)/.(int64)` 改读已转好的 `RuleTpl.Args`
- [ ] `Between` 签名 `(val any, min,max int64)` → `(val, min, max any)`，内部走 `valueCompare`（与 Gt/Lt 一致）；§5 记一条签名破坏
- [ ] **修 `val.(string)` panic**：`IsEmail/Regexp/IsJSON/IsStringNumber` 等对非字符串字段的分派改为安全转换或返回 false（不 panic）；补非字符串入参用例
- [ ] 评估是否一并把这些纯字符串验证器的内部分派统一（公共 `string` 签名可保留，分派层去断言）
- [ ] 受影响时拆分 `validators.go`（1305 行 > 1000）；否则该项留 V_cleanup

**坑（重点）**
- **golden 基本不变**：路线 2 保留签名、转换语义等价（只是搬到构建期），golden 的 `%#v` 应保持——若变则说明转换不等价，须排查。这与原"路线 1 必改 golden"截然不同。
- 构建期转换须保留 nil 语义、`*T(nil)` 与 `T` 零值区分（延续 v1.6.0 P3 克隆/nil 规则）；动态类型路径（slice/map-of-struct）的 args 转换时机须与静态模板一致。
- `convertArgsType` 删除后，确认其原有的「转换失败 → `convArgTypeError`」错误路径在构建期等价重建（构建期失败如何报错：建模板时即报 or 延迟到验证期）。
- panic 修复改变的是「非字符串字段 + 字符串验证器」这一**异常**分支（原 panic，现 false/error）——确认无测试依赖旧 panic 行为。

**验收门禁**：golden `count=3` 逐字节绿（路线 2 期望不变）；全量 + race 绿；`convertArgsType` 已删、其运行期分配清零，StructFlat allocs 较 v1.6.0 终态再降；新增非字符串入参用例验证不再 panic。
**提交**：`perf(rule): pre-convert rule args at build time, drop runtime convertArgsType`、`refactor(validators)!: unify Between to any params`、`fix(validators): avoid panic on non-string value for string validators`

### V4 — `DataFace` 扩展 + 消除源头 `Interface()` 装箱 🔴高风险

**目标**：`DataFace.TryGet` 返回携带 `reflect.Value` 的载体，消除 `data_source.go` 缓存命中后 `fv.Interface()` 拆箱（痛点 A 第 1 跳）；`GetWithDefault` 返回 `*fieldValue` 全管线流转，落地设计 §4.3 **完整版**。同时把 `BindStruct/SafeVal` 纳入 `DataFace` 接口。

**关键设计（见设计 §4.3 / §0.1 / §6.0）**
```go
// 现状 DataFace（data_source.go:57）
type DataFace interface {
    Type() uint8
    Src() any
    Get(key string) (val any, exist bool)
    TryGet(key string) (val any, exist, zero bool)   // ← 破坏点：改返回 reflect.Value 载体
    Set(field string, val any) (any, error)
    Create(err ...error) *Validation
    Validation(err ...error) *Validation
    // v2.0 新增（破坏点：自定义实现者须补齐）
    // BindStruct(ptr any) error
    // SafeVal(key string) (any, bool)
}
```
- `TryGet` 返回值携带 `reflect.Value`（如 `(*fieldValue, bool)` 或 `(reflect.Value, exist, zero)`），`StructData` 取字段时直接塞入已有的 `reflect.Value`，**不再 `Interface()` 拆箱**。
- `GetWithDefault` 改返回 `*fieldValue`（内部），公共 `Get/Safe` 签名保持（内部取 `.src`）。
- 三个数据源 `MapData`/`FormData`/`StructData` 适配新接口；map/form 源无源 `reflect.Value`，载体懒构造（与现状等价）。

**任务**
- [ ] 定型 `DataFace.TryGet` 新返回形态（载体类型 + nil/zero 语义）
- [ ] 三数据源实现新 `TryGet`：`StructData` 直供 `reflect.Value`；`MapData`/`FormData` 懒构造载体
- [ ] `BindStruct`/`SafeVal` 提升进 `DataFace` 接口，三源补齐实现
- [ ] `GetWithDefault` 返回 `*fieldValue`，贯穿管线（消除痛点 A 第 1、2 跳）
- [ ] 更新所有 `DataFace` 调用点 + 内部桥接（v1.6.0 P1 的 `vfv=nil` 回退点复核）
- [ ] race 测试 + 内存对比

**坑**
- 自定义 `DataFace` 实现者是受影响群体——§5 清单 + 迁移指南须给出"如何适配新接口"。
- typed-nil vs untyped-nil 语义（v1.6.0 P1 的 `nilRVal`/`Invalid` 处理）在新载体上须等价复刻。
- `Set` 路径 v1.6.0 保守未改（仍 `FieldByName` + nil 指针初始化）——本阶段视情况一并梳理或继续保守，按"有不确定就不动"。

**验收门禁**：golden `count=3` 绿；全量 + race 绿；StructFlat / Nested 较 v1.6.0 终态 allocs 再降（源头装箱消除可量化）。
**提交**：`refactor(datasource)!: TryGet returns reflect.Value carrier; promote BindStruct/SafeVal to interface`

### V5 — 注册自定义类型 🟡中风险

### V6 — opt-in Factory + sync.Pool（Struct/Map）🟢低风险

**目标**：落地 v1.6.0 延后的可选 `Factory` + pooled `Validation` + `Release()`（CHANGELOG V2-TODO 第 3 项；设计 §4.7）。`Val/Var` 的池化 v1.6.0 已做（`fix(value)` 67292f0），本阶段补 `Struct/Map` 工厂。

**关键设计（见设计 §4.7）**
```go
f := validate.NewFactory()
v := f.Struct(&user)   // 内部 pool 取 Validation + 命中 typeCache
defer v.Release()      // 归还前 Reset 全字段
v.Validate()
```

**任务**

**验收门禁**：default 基准零回退；Factory heavy-reuse 场景 allocs 显著下降；race + 串扰测试绿。

## 6. 风险与回滚

| 风险 | 阶段 | 缓解 / 回滚 |
|---|---|---|
| 构建期 args 转换与运行期 `convertArgsType` 不等价 | V3 | 路线 2 复用同一转换逻辑、golden 期望**不变**——一旦 golden 变即说明不等价，立即排查；构建期须覆盖原 in-place 改写的全部分支（变长/nil/类型不符报错） |
| `Between` 改 any 后边界比较行为变化 | V3 | 走与 Gt/Lt 同一 `valueCompare`；补 int/float/越界用例对照旧 int64 行为 |
| 字符串验证器 panic 修复误伤正常路径 | V3 | 只改非字符串**异常**分支；正常 string 字段路径不变；golden 守护 |
| `TryGet` 返回值改型遗漏 typed-nil / zero 语义 | V4 | 复刻 v1.6.0 P1 的 `nilRVal`/`Invalid` 处理；专项用例锁定 |
| 自定义 `DataFace` 实现者无声破坏 | V4 | §5 清单 + 迁移指南显式说明；CHANGELOG 醒目标注 |
| V5 自定义类型 API 设计不当致返工 | V5 | 开工前先写一页小设计评审；并发安全参照 `typeCache` |
| 池化 `Reset` 不彻底串数据 | V6 | opt-in 非默认；`Release` 前 Reset 全字段；`-race` 复现 |
| `/v2` sweep 漏改 import | V7 | 放靠后一次性 sweep；`go build ./...` 兜底；examples 独立 go.mod 单独处理 |

**回滚策略**：每阶段独立提交、独立可编译；出问题回滚到上一阶段末尾绿色提交。V1/V2/V6 可独立合并；V3/V4 是核心破坏对，建议成对验证后再继续。即使只完成到 V4，已具备 v2.0 的核心 API 精简价值。
